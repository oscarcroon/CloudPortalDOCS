# Discovery and inventory

With **discovery**, you find certificates that are already installed on your servers, and with the **inventory** you get a consolidated view of all the certificates the portal knows about – regardless of how they got there.

## Discovery – find existing certificates

Once you have an agent installed on a server, the portal can discover certificates that already exist there. The discovered certificates are listed under **Certificates → Discovered**.

### What the list shows

| Column | Content |
|--------|----------|
| **Host** | The server where the certificate was found. |
| **Subject** | The certificate's subject (usually the domain name). |
| **Thumbprint** | A shortened fingerprint that uniquely identifies the certificate. |
| **Expires** | The certificate's expiry date. |
| **Status** | Adoption state (see below). |

You can filter the list by state via the menu at the top.

### Adoption states

| State | Meaning |
|-----------|-----------|
| **Discovered** | Found on a server, no action taken yet. |
| **Matched** | Matches a certificate that already exists in the inventory. |
| **Adopted** | Taken over by the portal, which now manages it. |
| **Ignored** | Marked as not relevant. |

### Adopt a certificate

To **adopt** means the portal takes over management of the certificate, so it can be tracked and renewed automatically.

1. Find the certificate in the list (state **Discovered** or **Matched**).
2. Click **Adopt** on the row.
3. In the dialog you can **assign a profile** (optional). The profile controls rules such as renewal. Leave it empty if you don't want to link a profile yet.
4. Confirm by clicking **Adopt**.

::tip
Assign a profile at adoption if you want the certificate to be renewed automatically on the portal's terms. Without a profile, it's included in the inventory but not actively managed.
::

## Inventory – your certificate inventory

Under **Certificates → Certificates** you'll find the certificate inventory – a list of all the certificates the portal knows about.

### Search and filter

- **Search** – enter a domain in the search box to filter the list.
- **Status filter** – filter by status, for example active, expiring soon, expired, revoked, renewed, or replaced.

### Columns in the list

| Column | Content |
|--------|----------|
| **Domain** | The certificate's primary domain. |
| **Issuer** | Which certificate authority issued it. |
| **Thumbprint** | Shortened fingerprint. |
| **Status** | Current status, color-coded. |
| **Expires** | Expiry date. |
| **Source** | How the certificate entered the portal. |

### View details

Click a row to expand it. This shows, among other things:

- Full serial number and thumbprint.
- Installation target.
- Next scheduled renewal and renewal strategy.
- Whether the certificate is **managed** or **unmanaged**.
- A link to the order, if the certificate came from one.

::note
**Managed** means the portal actively handles renewal and tracking. **Unmanaged** means the certificate is only included for an overview – it won't be renewed automatically until it's adopted or linked to a profile.
::

## Import a certificate manually

Do you have details about a certificate but no agent to find it? Then you can register it manually.

1. Go to **Certificates → Certificates**.
2. Click **Import certificate**.
3. Fill in the form. Only the **primary domain** is required – the rest is optional:
   - SAN domains (comma-separated)
   - Serial number
   - Thumbprint
   - Issuer
   - Issued date and Expires date
4. Click **Import**.

The imported certificate is added to the inventory as unmanaged. If you want it to be renewed automatically, you need to link it to a profile.
