# Order a certificate

Here's how to order a new SSL/TLS certificate for one or more domains. You place the order through a multi-step wizard where you choose a profile, domains, a validation method, and where the certificate should be installed.

::note
Before you order your first certificate, you normally need at least one active credential set (the connection to the certificate authority) and a registered agent that can install the certificate. See [Deployment and agents](/en/certificates/deployment-agents).
::

## Start a new order

1. Go to **Certificates → Orders**.
2. Click **New order**.
3. The wizard opens on the first step. Click **Next** to move forward and **Back** to go back. You can also click an earlier step in the step indicator at the top.

## Steps in the order wizard

### 1. Profile (optional)

Choose a **certificate profile** if you want to. The profile contains ready-made rules for the certificate authority, allowed domains, and renewal. When you select a profile, several later choices are filled in automatically for you.

Choose **No profile** if you want to configure everything yourself in the following steps.

### 2. Credential set

Choose which **certificate authority** the order should go to. The list shows the active credential sets that exist, with their name and provider.

::note
If you selected a profile in step 1, the credential set may already be pre-filled.
::

### 3. Agent

Choose which **agent** should carry out the order and install the certificate on the server. Under each agent you'll see its capabilities (for example `iis`, `pfx`, `dns-01`) shown as tags.

::note
Agents that aren't active are not shown in the list. Missing an agent? See [Deployment and agents](/en/certificates/deployment-agents).
::

### 4. Domains

Enter the **primary domain** (for example `example.com`).

If the certificate needs to cover more domains, add **SAN domains** (Subject Alternative Names):

1. Click **+ Add SAN**.
2. Enter the domain name (for example `www.example.com`).
3. Repeat for additional domains. Remove a row with the cross button next to the field.

::tip
Wildcard domains such as `*.example.com` are supported, but they require DNS validation (DNS-01). In that case the wizard automatically selects the correct validation method for you in the next step.
::

### 5. Validation

This is where you prove that you own the domains. Choose a validation method:

- **HTTP-01** – a validation file is placed on the web server. Does not work for wildcard domains.
- **DNS-01** – a TXT record is added to the domain's DNS. Required for wildcards and works for all domains.

If you choose **DNS-01**, you also specify how the TXT record should be created:

- **Manual** – you create and remove the TXT record yourself.
- **Portal-managed zone** – the portal handles the TXT record for you. Select the DNS zone that matches your domain from the list.

::note
When the portal can match your domain to a known DNS zone, the wizard automatically suggests the most specific zone. You can change the selection manually if needed. If a DNS provider isn't configured, it's shown as unavailable.
::

### 6. Installation

Choose an **installation target** – where the certificate should end up once it's issued, for example IIS, a PFX file, or Central SSL. Choose **Manual** if you don't want the certificate to be installed automatically.

::note
Which installation targets you can choose depends on the selected agent's capabilities. Targets that the agent doesn't support are disabled.
::

Also configure automatic renewal:

- **Automatic renewal** – check this to let the portal renew the certificate automatically.
- **Days before expiry** – how many days before expiry the renewal should happen (1–90, default 30).

### 7. Review and submit

The final step shows a summary of everything you've chosen: credential set, agent, domains, validation method, any DNS zone, installation target, and renewal setting.

Check that everything is correct and click **Submit order**. You're then taken to the order's detail view where you can follow its progress.

## When the order requires approval

Some orders must be approved before the certificate is issued – for example orders with wildcard domains, or if the selected profile requires approval. In that case the order moves to the **Pending approval** state.

::warning
No work starts before the order is approved. No TXT record is created and no call is made to the certificate authority until an authorized administrator has approved the order.
::

You follow an order in **Certificates → Orders** and open the detail view to see its status and any approval requirements.

## After the order

Once the certificate is issued and installed, it appears in your inventory. There you'll see its status, expiry date, and next scheduled renewal. Read more under [Discovery and inventory](/en/certificates/discovery-and-inventory).
