# Connect GitHub

Before you can import and build an app in App Hosting, you connect the GitHub account or GitHub organization where the code lives. The connection is made by installing CloudPortal's GitHub App on the repositories you choose.

::note
You only connect the repositories you choose yourself. App Hosting never sees more than the repos you have granted the app access to.
::

## How to connect GitHub

1. Go to **App Hosting** and open the **Connect GitHub** page.
2. Click **Install GitHub App**.
3. You are sent to GitHub. Log in if you are not already signed in.
4. Choose which account or organization the app should be installed on.
5. Choose whether the app should have access to **all repositories** or just **selected repositories**, and mark the repos you want to be able to import.
6. Confirm the installation. You are automatically sent back to the portal, where the connection is completed.

Once the connection is complete, a confirmation is shown with the account name, account type and the number of connected repositories.

::tip
The easiest approach is to start from the **Connect GitHub** page — the connection is then filled in automatically and you can go straight to the import with the **Continue — import app** button.
::

## Connected GitHub accounts

On the **Connect GitHub** page you see a list of the accounts where CloudPortal's GitHub App is already installed. Each row shows the account name and a status label, for example:

- **Installed** — the connection is working.
- **Permissions missing** — the app is installed but needs more permissions (see below).
- **Suspended** — the installation is suspended on GitHub.
- **Repo removed** / **Uninstalled** — the connection needs to be restored.

The list only shows the repositories the app has been granted access to. To add more repos, use **Manage installation** to open the settings on GitHub.

### Add more repositories or accounts

- **Select more repos in GitHub** — opens the GitHub installation where you can expand which repos the app sees. Click **Refresh** in the portal afterward so the new repos appear.
- **Connect more GitHub accounts** — install the app on another account or organization.

## Permissions missing

If a connection shows **Permissions missing**, the portal lists exactly which permissions are needed. To build and publish apps, the app normally needs to be able to:

- write content in the repo (to add a workflow file),
- write files under `.github/workflows`,
- read and write the repo's Actions secrets.

Update the installation on GitHub with the correct permissions and try again.

::warning
If permissions are missing, the import and build cannot be completed. Fix the permissions on GitHub before proceeding.
::

## Disconnect an account

Each row has a **Disconnect** option. It uninstalls CloudPortal's GitHub App from that account on GitHub.

::warning
An account used by one or more apps in App Hosting cannot be disconnected. First remove the apps that use the connection, then disconnect the account.
::
