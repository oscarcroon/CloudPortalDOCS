# Redirects

With Redirects you set up HTTP redirects for a domain — for example to move visitors from an old address to a new one, or to forward an entire domain to another website.

## Choose a zone

1. Go to **Domains & DNS** and select **Redirects** (Manage redirects). You can also reach it via the **Redirects** button on the zone page.
2. Select the **zone** whose redirects you want to manage. Use the search field if you have many zones.
3. The redirect list for the zone opens and shows, among other things, host, path, target URL, type, status code and number of hits.

## Forward a domain (simplest)

Use **Forward domain** when you want to send an entire domain on to a new address.

1. Open the **Forward domain** tab.
2. Enter the **Destination URL** (where visitors should be sent), e.g. `https://newsite.com`.
3. Leave the domain field empty to forward the main domain (apex), or enter a prefix for a subdomain.
4. Under **Advanced** you can choose to also include `www`, and whether **paths should be preserved** (e.g. that `/about` carries over to the destination) or whether everything should be sent to the destination's root.
5. Click **Create forward**.

::note
For a forward to work, the portal may need to update the domain's DNS records so that they point to the redirect service. In that case a dialog is shown listing exactly which records are removed and created. Review and confirm. If you lack permission to change DNS, an administrator needs to enable the forward.
::

## Create a custom redirect

Use **Custom redirect** when you want to control individual paths.

1. Open the **Custom redirect** tab and click **Create redirect**.
2. Enter the **Host** — `@` or empty for apex (the zone's domain), or e.g. `www` for a subdomain.
3. Select the **Redirect type**:
   - **Simple** — exact path match, e.g. `/old-page`.
   - **Wildcard** — pattern with `*`, e.g. `/products/*`. A `*` in the target URL preserves what was matched.
   - **Regex** — regular expression, e.g. `^/blog/(.+)$`, with `$1`, `$2` etc. in the target URL.
4. Enter the **Source path** (must start with `/`) and the **Target URL** (a full `http://` or `https://` address).
5. Select the **Status code**:
   - **301 / 308** — permanent (301 is the most common for a domain move; 308 preserves the HTTP method).
   - **302 / 307** — temporary (307 preserves the HTTP method, good for forms).
6. Tick **Active** if the redirect should apply immediately.
7. Click **Create**.

::tip
The shortcut **Domain move (apex + www)** creates two redirects at once — both apex and `www` are pointed to the new domain.
::

## Manage existing redirects

In the list you can:

- **Edit** or **Delete** an individual redirect.
- **Enable** or **Disable** redirects.
- Select several and perform actions in bulk.
- **Filter** by type, status code and status (active/inactive), or search.
- **Import** redirects from a CSV or TXT file, and **Export** existing ones.

## Many redirects (RewriteMaps)

For larger migrations you can group rules in a **RewriteMap**. A RewriteMap is a draft that you can **publish** and **unpublish** as a group, which makes a large set of rules easier to manage. You find them under the **RewriteMaps** tab.

::note
Before you can publish a RewriteMap, any conflicts with other active redirects and duplicated paths need to be resolved. The portal shows what needs to be addressed.
::

## Park a domain

With **Park domain** you can show a generic "coming soon" page for a domain that does not yet have a real website. DNS is updated automatically and TLS is handled by the parking service.

## Downstream effects

::warning
When a redirect or forward requires DNS changes, existing records for the same name may be replaced. This can affect other traffic that uses the same DNS name. Always review the change dialog that is shown before you confirm.
::
