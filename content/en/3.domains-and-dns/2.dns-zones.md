# DNS zones

A DNS zone corresponds to a domain (for example `example.com`) and contains all DNS records for that domain. On the zones page you can see which zones are linked to your organization and open a zone to work with its records.

## View your zones

1. Go to **Domains & DNS** and select **DNS zones**.
2. The list shows the zones that belong to your organization. For each zone you see the domain name, which server it lives on and how many records the zone contains.
3. Use the **search field** at the top to filter by domain name.
4. Use the **sort menu** to sort by name (A→Z or Z→A) or by number of records (most/fewest).
5. Click the **refresh button** (the circular arrow) if you want to fetch fresh data.

::note
If no zones are shown, it may be that no zones have been activated for your organization yet. In that case the text "Contact an administrator to activate zones" is displayed. Contact your administrator or see [Autodiscover](/en/domains-and-dns/autodiscover) to find and activate zones.
::

## Open a zone

Click a zone in the list to open it. On the zone page you see:

- **Records** — the main workspace where you create and edit DNS records. See [DNS records](/en/domains-and-dns/dns-records).
- **Zone information (SOA)** — basic information about the zone, such as the primary name server, the responsible contact and the serial number.
- **Name servers** — the name servers that are authoritative for the zone.

::note
The name servers and the SOA record are managed automatically and cannot be changed through the portal.
::

### Domain delegation

For a zone to work, the domain must point to the correct name servers at your domain registrar. If the delegation is not complete, a marker indicating that action is required is shown, along with which name servers you need to add at your registrar.

::warning
When you change name servers at the registrar it can take up to 24 hours for the change to take effect (propagation), often sooner.
::

## Cloudflare zones

Zones hosted on Cloudflare are managed under **Cloudflare DNS**. The interface works largely the same way: you see a list of zones, can search and open a zone to manage its records.

Differences to be aware of for Cloudflare zones:

- A zone can have the status **pending** until the domain has been pointed to Cloudflare's name servers. The name servers are shown once the zone has been created.
- Records can be **Proxied** (traffic goes through Cloudflare) or **DNS only** (Cloudflare only answers DNS, no proxying).

::note
To manage Cloudflare zones, your organization needs to have an API token linked. It is configured by an administrator. If you lack access, contact your administrator.
::

## Export and import a zone

If you have permission, on the zone page you can:

- **Export** the zone to a text file with all records — useful as a backup.
- **Import** records from a zone file. The portal parses the file and shows a preview of what will be imported, skipped or flagged with an error before you confirm.

::note
Some records are never imported because they are managed automatically, including the SOA record, the zone's name servers and DNSSEC records.
::

## Delete a zone

Deletion is found under **Danger zone** on the zone page and requires special permission. You must type the zone's name and confirm that the action cannot be undone.

::warning
Deleting a zone permanently removes the zone and all of its DNS records. This cannot be undone.
::
