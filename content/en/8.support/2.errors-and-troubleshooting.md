# Errors and troubleshooting

This page collects the most common problems in the portal and how to resolve them step by step. If you can't find the answer here, continue to [Contact support](/en/support/contact-support).

::tip
Before troubleshooting more deeply: reload the page and check [Service status](/en/support/service-status). Sometimes an error is caused by ongoing maintenance or a known service disruption.
::

## Sign-in

### Incorrect email or password

If you get the message **"Incorrect email or password"**:

1. Check that the email address is spelled correctly and has no extra spaces.
2. Check that Caps Lock is not on.
3. If you have forgotten your password, select **Forgot password?** on the sign-in page and follow the instructions.

When you request a new password, a link is sent to your email if the address exists in the system. Follow the link in the email to choose a new password.

::note
For security reasons, the portal shows the same confirmation regardless of whether the address exists or not. If you don't receive an email, check your spam folder and make sure the address is correct.
::

### The organization requires SSO

If you get the message that **the organization requires SSO**, you should not sign in with email and password. Instead, use the SSO button on the sign-in page to sign in through your organization's identity provider.

### Two-factor code (MFA) doesn't work

If the verification code is rejected:

1. Check that the clock on the phone where your authenticator app is installed is set correctly (preferably automatic time). One-time codes are time-based.
2. Wait until a new code is generated and try it.
3. If you can't access the authenticator app, you can use one of the backup codes you received when you enabled two-factor authentication.

::warning
If you have lost both the authenticator app and the backup codes, you cannot reset two-factor authentication yourself. Contact your provider or organization administrator for help.
::

### Verification failed

If you get **"Verification failed"** when signing in, it is usually caused by the bot protection (Turnstile). Reload the page and try again. Blocking browser extensions or a very restrictive firewall can interfere with the verification.

### "No access context could be determined"

This means you are signed in but have not yet been linked to any organization or tenant. Contact your administrator so they can grant you access.

## I don't see what I expect

### A menu, module, or button is missing

The portal only shows what you have permission to access. If a feature is missing, it is usually due to your role:

1. Check that you are in the right context (organization/tenant) – see the next section.
2. Request additional permissions from your administrator if you need access to a module.

### The wrong organization or tenant is shown

The portal is multi-user and multi-tenant. Use the context switcher (at the top of the interface) to switch to the right organization:

1. Open the context switcher.
2. Search for the organization and select it.
3. The view reloads in the selected context.

::note
If you get **"The organization requires SSO"** when switching, you must start the SSO flow to continue into that context.
::

### Data doesn't load / the page is empty

1. Click **Refresh** or reload the page.
2. Check your internet connection.
3. If the problem persists, check [Service status](/en/support/service-status) for ongoing disruptions.

## DNS and domains

When you make changes in DNS, the portal publishes the records to the provider, but changes don't take effect immediately everywhere.

::warning
The portal publishes the DNS record, but it is the receiving servers and resolvers that determine when they fetch the new information. Expect propagation time before the change is visible to everyone. In addition, security features in other systems decide for themselves whether they honor a published record.
::

If a change doesn't seem to take effect:

1. Check that you saved the change and that the record appears in the list.
2. Wait a while and check again – propagation takes time.
3. Verify that you edited the correct zone/domain.

## Email and invitations

- **The invitation or reset email didn't arrive** – check your spam folder and that the correct address was used. Ask your administrator to resend the invitation if it has already expired.
- **The link in the email no longer works** – links for invitations and password resets are time-limited. Request a new one.

## Notifications and service messages

Notifications and service messages are shown in the portal. If you have hidden a service message, you can show it again via the service message settings ("Show again").

## General error messages

::tip
When you report an error to support, include:

- The exact error message (a screenshot is ideal).
- Which page or module you were on.
- Which organization/tenant you were in.
- The approximate time.
::

::note
Screenshot: an error message in the portal with the page address visible in the browser's address bar.
::
