# SCIM provisioning

SCIM lets your identity provider (e.g. Microsoft Entra ID) **create and update users automatically** in CloudPortal and **assign roles via groups**. SSO handles signing in — SCIM handles which accounts exist and what they can do.

::note
SCIM requires [SSO](/en/organization/sso) to be set up first: at least one verified domain and an active identity provider. The SCIM section appears under **Settings → Authentication** once that's in place. Partners do the same under **Partner → Tenants → [customer] → Authentication**.
::

## Enable SCIM

1. Go to **Settings → Authentication → SCIM provisioning**.
2. Click **Generate new token**. The portal shows a **bearer token once** — copy it immediately (it can't be shown again).
3. Note the **Tenant URL** the portal displays:
   ```
   https://<your-domain>/api/scim/v2
   ```

## Connect it in your identity provider

In Entra, for example (Enterprise Application → Provisioning):

1. Set **Provisioning Mode** to *Automatic*.
2. Under **Admin Credentials**:
   - **Tenant URL** → paste `https://<your-domain>/api/scim/v2`
   - **Secret Token** → paste the token from the portal
3. Click **Test Connection** to verify.
4. Enable group provisioning and assign the users/groups to sync.
5. Start provisioning.

::note
The first sync can take a while (often around 40 minutes, depending on your provider). Synced groups appear in the portal but grant **no access** until you map them to roles (below).
::

## Map groups to roles

In the portal's **group-role mappings** section, link a synced group to permissions:

1. Pick a group and click **Add mapping**.
2. Choose the type:
   - **Organization role** — admin, operator, member, viewer or support
   - **Module bundle** — a combined level (e.g. Admin/Operator/Viewer) for a module
   - **Single module permission** — fine-grained permission (advanced)
   - **Provisioning** — presence only, no permissions
   - **MSP role** — partner scope only
3. Choose **what happens when a user is removed from the group**: remove the grant (recommended), downgrade to the default role, or no action.
4. Save. Users in the group get the role **immediately** — no re-sync needed.

::tip
Partners mapping at the tenant level can tick **Applies to all child organizations** to let a mapping take effect across all of the customer's organizations.
::

## Manage tokens

The token list shows prefix, created date, last used and status. You can **revoke** individual tokens, or **disable SCIM** entirely (which revokes all tokens).
