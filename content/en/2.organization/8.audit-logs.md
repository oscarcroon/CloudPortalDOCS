# Audit logs

The audit log shows a history of important events in your organization — for example when someone is invited, changes role or a setting is changed. It provides traceability over who did what and when.

::note
To view the audit log you need permission to read logs. If you lack access, contact an administrator or owner in the organization.
::

## Reading the log

Each row in the log shows:

- **Time** — when the event occurred.
- **Event** — which type of event it concerns.
- **User** — who performed the action.
- **Details** — click **View** to see more information about an individual event.

The log is shown page by page. Use **Previous** and **Next** at the bottom to page through it.

## Filtering events

At the top there are filters to help you find the right events:

1. **Module** — narrow down to a specific part of the portal.
2. **Event type** — choose a specific type of event (the list adapts to the selected module).
3. **From date** and **To date** — limit to a time period.

The log updates automatically when you change a filter. Click **Clear** to return to the default view.

::tip
By default the most recent events are shown. If you want to investigate something that happened a while ago, set **From date** and **To date** so they cover the right period.
::

## What the log shows and does not show

The audit log is intended to give insight into which actions have been performed. It lists events, identifiers and status — not the full content of what was changed.

::note
The audit log is intended for follow-up and review, not as a complete backup or restore tool. If you need to restore something, do so through the corresponding feature in the portal.
::
