# Distribution och agenter

En **agent** är en liten bakgrundstjänst som körs på din server. Den tar emot certifikatbeställningar från portalen, sköter valideringen och installerar certifikatet lokalt. Med en agent på plats kan portalen distribuera och förnya certifikat automatiskt – utan att du loggar in på servern varje gång.

::note
Förutsättningar för en agent: Windows Server 2016+ eller Windows 10+, administratörsrättigheter på servern och utgående HTTPS-åtkomst till portalen.
::

## Registrera en ny agent

1. Gå till **Certifikat → Agenter**.
2. Klicka på **Registrera agent**.
3. Fyll i formuläret:
   - **Namn** – ett tydligt namn, till exempel `web-server-01`.
   - **Beskrivning** (valfritt).
   - **Kapabiliteter** – bocka i vad servern ska kunna hantera, till exempel HTTP-01, DNS-01, IIS, PFX eller Central SSL.
4. Klicka på **Registrera agent**.

::warning
Direkt efter registreringen visas agentens token en gång. Kopiera och spara den säkert – den visas inte igen. Behöver du en ny token senare använder du **Generera ny token**, men då slutar den gamla att fungera direkt.
::

## Installera agenten på servern

När du har registrerat agenten öppnas en **installationsguide**. Du når den även senare via knappen **Installationsguide** på agentens rad i listan. Guiden leder dig genom installationen steg för steg:

1. **Agenttoken** – kopiera token från guiden. Den behövs för att agenten ska kunna ansluta till portalen.
2. **Ladda ner och installera** – klicka på **Ladda ner agent** för att hämta agentpaketet, och **Ladda ner installationsskript** för PowerShell-skriptet som automatiserar installationen.
3. **Kör installationsskriptet** – extrahera den nedladdade filen och kör skriptet i PowerShell **som administratör**. Skriptet kopierar filer, skriver konfigurationen, skapar Windows-tjänsten och startar den. Det finns även ett läge för manuell konfiguration för den som vill ställa in allt själv.
4. **Verifiera** – guiden visar en liveindikator. När tjänsten är igång ska agenten visas som **online** inom ungefär 60 sekunder, med senast sedd IP och tidpunkt.

::tip
Agenten ansluter alltid utåt mot portalen. Dina servrar behöver inte vara åtkomliga utifrån – det räcker att de kan nå portalen via HTTPS.
::

## Se status för dina agenter

På sidan **Certifikat → Agenter** ser du varje agent med:

- Namn och status (aktiv/inaktiv).
- En **online**-markering med pulserande prick när agenten nyligen har hört av sig, annars **offline**.
- Senaste heartbeat och senast sedda IP.
- Agentens kapabiliteter som taggar.

En agent räknas som online när den hört av sig under de senaste minuterna. Hör den inte av sig markeras den som offline.

## Distribuera certifikat med en agent

När du beställer ett certifikat väljer du vilken agent som ska utföra det och vilket installationsmål som gäller (se [Beställ certifikat](/sv/certifikat/bestall-certifikat)). Agenten installerar sedan certifikatet på rätt plats och rapporterar tillbaka till portalen.

Du kan använda samma certifikat på flera servrar – varje installation följs upp separat, så att portalen kan visa var certifikatet faktiskt sitter och om något har ändrats. Sådana avvikelser följs upp under [Discovery och inventory](/sv/certifikat/discovery-och-inventory).

::note
Portalen ber agenten installera certifikatet, men det är servern och dess konfiguration som avgör om certifikatet verkligen tas i bruk. Kontrollera tjänsten på servern om en installation inte syns i portalen.
::

## Felsökning

| Problem | Att kontrollera |
|---------|-----------------|
| Agenten visas inte som online | Att CertAgent-tjänsten körs på servern. |
| Agenten når inte portalen | Att servern kan nå portalen via utgående HTTPS. |
| Anslutningen avvisas | Att rätt agenttoken är konfigurerad. Generera en ny token vid behov. |
| Installationen ger oväntat fel | Agentens loggar på servern. |

::note
Skärmdump: Agentlistan med en agent markerad som online, dess kapabilitetstaggar och knappen Installationsguide.
::
