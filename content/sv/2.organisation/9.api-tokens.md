# API-tokens

API-nycklar ger program och integrationer åtkomst till din organisations data via portalens API — till exempel för automatiserade flöden, CI/CD eller externa system. Varje nyckel får bara de behörigheter du väljer.

::note
För att skapa eller återkalla API-nycklar behöver du behörighet att hantera organisationen. Saknar du tillgång visas en informationsruta och knapparna är inaktiverade.
::

## Översikt över nycklar

Listan visar för varje nyckel:

- **Prefix** — en kort, igenkännbar början på nyckeln (hela nyckeln visas aldrig igen efter skapandet).
- **Beskrivning** — din egen text om vad nyckeln används till.
- **Behörigheter (scopes)** — vad nyckeln får göra.
- **Skapad** och vem som skapade den.
- **Senast använd**.
- **Status** — Aktiv, Utgången eller Återkallad.

## Skapa en API-nyckel

1. Klicka på **Skapa ny API-nyckel**.
2. Ange en **beskrivning** (valfritt men rekommenderat, till exempel "CI/CD-pipeline" eller "Extern integration").
3. Välj **behörigheter (scopes)**. Du kan kryssa i enskilda behörigheter eller använda en av snabbmallarna för att fylla i en uppsättning på en gång.
4. Ange ett **utgångsdatum** (valfritt). Lämnar du fältet tomt skapas en nyckel utan utgångsdatum.
5. Klicka på **Skapa nyckel**.

::warning
Nyckeln visas **endast en gång**, direkt efter att den skapats. Kopiera den och spara den på ett säkert ställe innan du stänger rutan. Tappar du bort den måste du skapa en ny.
::

::tip
Ge varje integration en egen nyckel med bara de behörigheter den behöver, och en beskrivande text. Då blir det lätt att se vad en nyckel används till och att återkalla rätt nyckel om något behöver dras tillbaka.
::

## Använda nyckeln

Skicka med nyckeln i anropets autentiseringshuvud:

```
Authorization: Bearer <din-nyckel>
```

## Återkalla en API-nyckel

1. Hitta nyckeln i listan.
2. Klicka på **Återkalla** på nyckelns rad.
3. Bekräfta i dialogrutan.

::warning
Återkallning kan inte ångras, och en återkallad nyckel kan inte återaktiveras. Alla integrationer som använder nyckeln slutar fungera direkt — uppdatera dem med en ny nyckel innan du återkallar den gamla.
::

## Goda vanor

- Sätt ett **utgångsdatum** på nycklar som bara behövs en begränsad tid.
- **Återkalla** nycklar som inte längre används.
- Dela aldrig en nyckel i e-post, chatt eller kod som checkas in i ett repo.
- Använd kolumnen **Senast använd** för att upptäcka nycklar som kan tas bort.
