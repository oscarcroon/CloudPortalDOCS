# Koppla GitHub

Innan du kan importera och bygga en app i App Hosting kopplar du det GitHub-konto eller den GitHub-organisation där koden ligger. Kopplingen görs genom att installera CloudPortals GitHub App på de repositorier du väljer.

::note
Du kopplar bara de repositorier du själv väljer. App Hosting ser aldrig mer än de repos du gett appen åtkomst till.
::

## Så här ansluter du GitHub

1. Gå till **App Hosting** och öppna sidan **Anslut GitHub**.
2. Klicka på **Installera GitHub App**.
3. Du skickas till GitHub. Logga in om du inte redan är inloggad.
4. Välj vilket konto eller vilken organisation appen ska installeras på.
5. Välj om appen ska få åtkomst till **alla repositorier** eller bara **utvalda repositorier**, och markera de repos du vill kunna importera.
6. Bekräfta installationen. Du skickas automatiskt tillbaka till portalen, där anslutningen slutförs.

När anslutningen är klar visas en bekräftelse med kontonamn, kontotyp och antalet anslutna repositorier.

::tip
Smidigast är att utgå från sidan **Anslut GitHub** — då fylls kopplingen i automatiskt och du kan gå direkt vidare till importen med knappen **Fortsätt — importera app**.
::

## Anslutna GitHub-konton

På sidan **Anslut GitHub** ser du en lista över de konton där CloudPortals GitHub App redan är installerad. Varje rad visar kontots namn och en statusetikett, till exempel:

- **Installerad** — anslutningen fungerar.
- **Behörigheter saknas** — appen är installerad men behöver fler behörigheter (se nedan).
- **Pausad** — installationen är pausad på GitHub.
- **Repo borttaget** / **Avinstallerad** — kopplingen behöver återställas.

Listan visar bara de repositorier appen fått åtkomst till. Vill du lägga till fler repos använder du **Hantera installationen** för att öppna inställningarna på GitHub.

### Lägg till fler repositorier eller konton

- **Välj fler repos i GitHub** — öppnar GitHub-installationen där du kan utöka vilka repos appen ser. Klicka på **Uppdatera** i portalen efteråt så att de nya repona dyker upp.
- **Anslut fler GitHub-konton** — installera appen på ytterligare ett konto eller en organisation.

## Behörigheter saknas

Om en anslutning visar **Behörigheter saknas** listar portalen exakt vilka behörigheter som behövs. För att bygga och publicera appar behöver appen normalt kunna:

- skriva innehåll i repot (för att lägga in en workflow-fil),
- skriva filer under `.github/workflows`,
- läsa och skriva repots Actions-hemligheter.

Uppdatera installationen på GitHub med rätt behörigheter och försök igen.

::warning
Saknas behörigheter kan importen och bygget inte slutföras. Åtgärda behörigheterna på GitHub innan du går vidare.
::

## Koppla bort ett konto

På varje rad finns **Koppla bort**. Den avinstallerar CloudPortals GitHub App från det kontot på GitHub.

::warning
Ett konto som används av en eller flera appar i App Hosting kan inte kopplas bort. Ta först bort apparna som använder kopplingen, koppla sedan bort kontot.
::
