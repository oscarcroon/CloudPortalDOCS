# Secrets och miljöer

Miljövariabler och hemligheter styr hur din app kopplar mot databaser och externa tjänster. I App Hosting hanterar du dem i importguidens steg **Secrets / miljövariabler** och senare på appens sida.

::note
Att lägga in värden är rekommenderat men inte obligatoriskt för att gå vidare — du kan komplettera senare.
::

## Publika kontra server-värden

App Hosting skiljer på två typer av värden:

- **Publika / frontend** — värden som börjar på till exempel `VITE_` eller `NEXT_PUBLIC_`. De byggs in i appen vid bygget och är synliga i webbläsaren.
- **Server / runtime** — hemligheter som bara används på servern och aldrig exponeras i frontend.

::warning
Frontend-värden är publika i den färdiga appen — behandla dem aldrig som hemligheter. Lägg endast hemliga nycklar bland server-värdena.
::

## Lägg till en miljövariabel

1. Öppna sektionen **Miljövariabler** (i importguiden eller på appens sida).
2. Klicka **Lägg till secret**.
3. Ange **Nyckel** och **Värde**.
4. Välj **scope** — publik/frontend eller server/runtime.
5. Spara.

Saknade värden som appen behöver markeras tydligt så att du ser vad som återstår.

## Vanliga variabler

Portalen ger vägledning för vanliga värden. Några exempel:

- **Supabase URL** — ditt Supabase-projekts API-URL (Project Settings → API → Project URL).
- **Supabase anon-/publishable-nyckel** — den publika nyckeln, normalt ofarlig i frontend (Project Settings → API → anon/public).
- **Supabase service_role-nyckel** — full admin-åtkomst. Endast server, aldrig i frontend.
- **Databas-anslutning** — endast server.
- **Stripe** — använd den publika publishable-nyckeln i frontend och den hemliga nyckeln endast på servern.

::warning
Supabase service_role-nyckeln ger full åtkomst till din databas. Den får aldrig ligga bland frontend-värdena.
::

## Miljöer (test och produktion)

Vid importen väljer du om appen ska ha både **Test + Produktion** eller **Endast Produktion**. En testmiljö låter dig bygga vidare i ditt verktyg utan att testa direkt i skarp drift. Du kan lägga till en testmiljö senare om du börjar med endast produktion.

::note
Miljövariabler hör till en specifik miljö. Värden du lägger in för test påverkar inte produktionen och tvärtom.
::
